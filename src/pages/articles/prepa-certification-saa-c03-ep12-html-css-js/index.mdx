Pour faire simple, lorsque vous tapez dans votre navigateur *www.amazon.fr,* les étapes suivantes se déroulent en arrière-plan : 

1. Le navigateur, Google Chrome par exemple, envoie votre requête, votre demande par votre connexion Internet. Votre requête doit arriver sur les machines d’Amazon. Les machines d’Amazon doivent recevoir votre requête. Ce sont ces machines qu’on appellent serveurs, les serveurs d’Amazon car un serveur est là pour servir, servir les réponses aux demandes, aux requêtes des navigateurs qu’on appellent dans le jargon “clients”. Ce jargon a été inspiré par ce qui se passe dans les restaurants avec le client qui émet sa demande, passe commande et le serveur qui sert, répond à la demande.
2. Seulement, lorsque vous téléphonez à votre ami, vous avez son numéro dans votre annuaire, un numéro attribué par l’opérateur téléphonique. Il est unique et personne d’autre ne peut décrocher ce numéro à part lui. Donc, au niveau de l’opérateur téléphonique, il y a un annuaire général où figure, pour chaque personne, un ou des numéros de téléphones. C’est la même chose avec Internet. Lorsque vous tapez *www.amazon.fr,* le navigateur a recours à ce qu’on appelle le DNS (**D**omain **N**ame **S**ystem) qui est une sorte d'annuaire pour sites web. Lorsque vous saisissez une adresse dans le navigateur, ce dernier consulte le DNS pour trouver l'adresse réelle du site web avant de la récupérer. C’est pour cela que le DNS est traduit par serveur de noms de domaines. Le nom de domaine correspond à ce que vous tapez dans le navigateur, *www.amazon.fr.* Le DNS est un serveur, une machine qui a la liste des adresses web, un peu comme l’annuaire téléphonique. Le navigateur a besoin de savoir sur quel serveur, quel machine le site web est situé pour pouvoir envoyer des requêtes au bon endroit.
3. Si le serveur accepte la requête émise par le client, le serveur envoie un message au client qui signifie : « *Pas de problème, tu peux consulter ce site web, le voici* ». Ensuite le serveur commence à envoyer les fichiers du site web au navigateur sous forme d'une série de petits morceaux nommés "paquet”. Ces petits paquets arrivent par votre connexion Internet. C’est pour cela que lorsque votre connexion internet est lente, vous voyez le site web apparaître progressivement. 
4. Le navigateur assemble les différents morceaux pour recomposer le site web en entier puis l'affiche sur votre écran.

Ces fichiers, ces petits paquets que le serveur envoie, vous pouvez le voir en faisant un clic droit avec votre souris et en choisissant “*Afficher le code source de la page*”… Faisons le avec [le modèle](https://tailwindui.com/templates/spotlight) que nous avons choisi de reproduire pour ce cours puisque notre approche pédagogique est orientée “projet”.

Ces messages cryptiques ont été envoyés par le serveur et décryptés par votre navigateur, vous avez le site web affiché. Ces messages cryptiques, c’est évidemment du code, du code que seul votre navigateur peut déchiffrer mais ce code a été écrit par des développeurs. Et c’est ce que nous allons faire à l’aide de trois (3) langages qui forment la base du développement web : le HTML, le CSS et Javascript. C’est à l’aide de ces trois (3) langages que les développeurs d’Amazon ont écrit les instructions que le serveur envoie au navigateur qui les lit et affiche le site comme ces instructions le demandent.

Pour reprendre le site “[InternetingisHard](https://www.internetingishard.com/html-and-css/introduction/)”…

> *HyperText Markup Language* (**HTML**), *Cascading Style Sheets* (**CSS**), et **JavaScript** sont les langages qui animent le web. Ils sont fortement très liés, mais ils ont été conçus pour des tâches très spécifiques. Comprendre comment ils interagissent est le chemin pour devenir un développeur web. Nous allons développer cela tout au long de ce tutoriel mais l'essentiel est ceci :
> 
> - HTML est fait pour ajouter du sens au contenu brut en le balisant,
> - CSS est fait pour le formatage (mise en page) du contenu balisé,
> - Javascript est fait pour rendre ce contenu et ce formatage interactifs.
> 
> Pensez au HTML comme le texte abstrait et les images derrière une page web, le CSS comme la page qui est actuellement affichée et Javascript comme les comportements qui peuvent manipuler à la fois le HTML et le CSS.
> 

 <Image src={htmlCSS} alt="html css js" />

Comme votre navigateur ne comprend que les instructions en HTML, CSS et Javascript, vous aurez besoin de maîtriser ces 3 langages pour pourvoir le transmettre les instructions pour qu’il affiche votre site web.

A titre d’information, il faut savoir que ces langages sont standardisés. Quelque soit le navigateur que vous utilisez, que ce soit Chrome, Firefox, Opera, Edge, Vivaldi, Brave ou autres, l'affichage sera pareil. Pourquoi ? parce que les standards existent. Pour faire simple, ce sont des normes respectées par tous les "*fabricants*" de navigateurs et que les développeurs doivent aussi respecter…

Pour y jeter un coup d’oeil : 

- le [standard HTML](https://html.spec.whatwg.org/multipage/) : [https://html.spec.whatwg.org/multipage/](https://html.spec.whatwg.org/multipage/)
- le [standard CSS](https://www.w3.org/Style/CSS/) : [https://www.w3.org/Style/CSS/](https://www.w3.org/Style/CSS/)
- le [standard Javascript](https://tc39.es/ecma262/) : [https://tc39.es/ecma262/](https://tc39.es/ecma262/)

Voilà pour la présentation des trois langages principaux du développement web. 

De la même manière que lors de votre cycle primaire, on vous apprenait les lettres, les syllabes et les phrases, c’est le même processus que vous allez subir avec son lot de difficultés, d’efforts, d’incompréhensions et autres. Et c’est tout à fait normal puisque c’est un nouveau langage, un nouveau monde que vous découvrez. Donc ne pas comprendre au premier coup ne veut pas dire que vous n’êtes pas intelligent ou que la “programmation n’est pas faite pour moi” comme nombre de personnes le disent. C’est juste le processus normal d’apprentissage.
